XML file format was chosen for the convenience of grouping multiple data representation for each object. The downside of it, is the slowness to load that data at start. Therefore, we enable the user to convert the list of vertices and faces from the XML file to binary file that is faster to read. In addition, instead of computing the normals per vertex each time the data is loaded, we compute them once and later require only the directory for the exra file path to access these files.

As the Astrocyte and Neurites input files are separated (objects vertices and faces for each file is independent), the exra files (including vertices and faces and normals as binary file) are computed separately for each.

  1. Checking "Compute Astrocyte Normals + Compressed Binary" will produce the following files in the "extra files path":
    • astro_normals.dat
    • astro_data_fv.dat
  2. Checking "Compute Neurites Normals + Compressed Binary" will produce the following files in the "extra files path":
    • neurites_data_fv.dat
    • neurites_normals.dat

Finally, the user can specify the number of objects to load using "Objects Limit".
